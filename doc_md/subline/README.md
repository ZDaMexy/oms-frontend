# Frontend Subline

`subline/` 用于记录前端主线之外的并行支撑工作，目标是服务主线，而不是替代主线。

当前不再直接在 `subline/` 根目录平铺五大文档，而是按功能方向进一步收束成多个子目录。每个子目录独立维护自己的 `README.md`、`constraints.md`、`dev-plan.md`、`dev-progress.md`、`changelog.md`。

## 适用内容

- 视觉探索、动效方向与品牌表达试验
- 页面文案草稿、FAQ 草案、多语言预案
- SEO、可访问性、部署辅助脚本等不阻塞主线的工作

## 当前子目录

- `P1-A/`：Phase 1 的首个支线占位，预留宣传展示阶段的单一功能方向拆分方式

## 使用原则

- 支线结果必须可丢弃、可替换或可平滑并入主线
- 一旦某项支线结果被采纳，必须同步更新主线文档
