# Frontend Mainline Plan

## 产品阶段

1. Phase 1：宣传展示
2. Phase 2：账号系统
3. Phase 3：个人主页与成绩展示
4. Phase 4：排行榜
5. Phase 5：谱面下载入口与私服相关服务

## 阶段计划

### Phase 1：宣传展示

- 完成仓库初始化、忽略规则和正式 README
- 建立 `doc_md/` 的主线、支线、调研线文档框架
- 将 `subline/` 从平铺文档收束为按功能方向拆分的目录结构，并先建立 `P1-A/` 占位支线
- 建立工作区根目录 `dev_bridge_md/`，为未来前后端对接预留统一桥文档落点
- 规划官网的信息架构、页面栏目与后续扩展位
- 实现静态宣传展示页面并准备 Nginx 部署

### Phase 2：账号系统

- 为注册、登录、找回密码、账户设置预留入口页与导航结构
- 对接后端认证接口并形成基础账户体验

### Phase 3：个人主页与成绩展示

- 提供玩家主页、成绩列表、单曲表现与 EX-SCORE 展示页面
- 打通客户端与 Web 的成绩展示入口

### Phase 4：排行榜

- 提供全站排行榜、曲目榜单、难度表关联榜单等视图
- 为排序、筛选和用户跳转设计稳定页面结构

### Phase 5：谱面下载入口与私服相关服务

- 提供谱面下载入口页、下载说明与服务入口页
- 与后端和本地服务器协同承接大文件下载、私服相关能力与账户联动

## 当前最近任务

- 保持 Phase 1 的宣传展示目标不偏离完整产品路线
- 固定支线目录收束方式，避免后续支线文档继续无序平铺
- 固定前端文档与桥文档的联动规则，避免跨边界议题只记录在单边仓库
- 先完成静态展示期的文档与信息架构，再进入首版页面实现
