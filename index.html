Current posture

静态宣传页先把事实说准，再给完整 Web 产品留出入口。

当前这版不是“官服已上线”的包装页，而是一个明确标识现状、能力方向和未来入口结构的公开门面。

Safe public facts

  • 默认构建关闭在线功能，默认 endpoint 为空。
  • 客户端内部保留账号、聊天、好友、观战、多人和元数据入口。
  • BMS 已是正式 ruleset，但在线 score 流程仍需额外改造。

Project shape

OMS 不是一次性的落地页，而是完整客户端、私服和 Web 入口的公共外壳。

Client

Windows 桌面优先

当前公开口径聚焦桌面客户端，避免给出误导性的跨平台承诺。

Ruleset

BMS 原生接入

项目围绕 BMS 模式、原生解析、keysound 与独立判定体系构建。

Server

在线能力先收边界

客户端有联机接点，但当前发行配置还没有接入 OMS 私服的可用远端。

Web

为未来入口留位

账户、主页、成绩、排行榜、谱面数据库和多人入口都在这套结构里提前预留。

Build state

默认离线优先

当前构建默认关闭在线功能，默认 endpoint 配置为空，不应把现状描述成“已经可直接连上 OMS 私服”。

Update channel

游戏内更新默认关闭

当前桌面端没有启用游戏内更新通道，因此下载与版本说明页要走明确的外部发布路径。

Online surfaces

在线接点仍保留

账号、聊天、好友 presence、观战、多人、元数据、分数提交和回放下载等入口都还在客户端内部。

Ruleset identity

BMS 已经是正式 ruleset

当前可安全公开的规则集事实是：BMS 已存在独立玩法与配置体系，短名为 bms。
